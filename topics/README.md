# Topics

Cross-cutting knowledge that doesn't belong to any one project. The first
tier of the wiki is per-project `AGENTS.md`; this directory is the second
tier.

See [`../spec/topics.md`](../spec/topics.md) for:

- when to write a topic vs a per-project gotcha
- topic file structure (frontmatter, sections, backlinks block)
- the promotion workflow (gotcha → topic, when seen in 2+ projects)
- naming and scope conventions

This directory is currently empty — topics get added as cross-cutting
knowledge surfaces. Add one when a piece of knowledge applies to multiple
projects or describes a *technology* / *pattern* rather than specific code.
